# AI/ML 개발자를 위한 수학-통계학 용어 정리

부제: 수학/통계학 비전공 IT 개발자 관점에서 이해하는 AI, 머신러닝 핵심 개념

---

## 1. 이 문서의 목적

이 문서는 수학이나 통계학을 전공하지 않은 IT 개발자가 AI(인공지능), ML(머신러닝), DL(딥러닝) 문서를 읽을 때 자주 마주치는 용어를 쉽게 이해할 수 있도록 정리한 실무형 가이드다.

핵심 목표는 다음과 같다.

- 어려운 수학 용어를 한글, 한자, 영어로 함께 이해한다.
- 용어의 원래 뜻보다, 개발자가 코드와 시스템 관점에서 어떻게 해석하면 되는지 익힌다.
- AI/ML에서 수학이 실제로 어떤 역할을 하는지 흐름으로 이해한다.
- 수식이 나와도 겁먹지 않고, "이건 결국 오차를 줄이기 위한 계산"이라는 큰 그림을 잡는다.

이 문서는 엄밀한 수학 교과서가 아니라, 개발자가 빠르게 감을 잡기 위한 "개념 번역서"에 가깝다.

## 2. AI/ML에서 수학은 무엇인가

AI/ML에서 수학은 정답을 외우는 학문이라기보다, 다음을 숫자로 다루는 도구다.

1. 현실의 데이터를 숫자로 표현한다.
2. 입력과 출력의 관계를 식으로 표현한다.
3. 얼마나 틀렸는지 계산한다.
4. 덜 틀리도록 수정 방향을 계산한다.

즉, AI/ML의 핵심은 다음 한 문장으로 요약된다.

> 모델은 식이고, 학습은 그 식의 숫자들을 조정해서 오차를 줄이는 과정이다.

예를 들어:

- 이미지 -> 픽셀 숫자 배열
- 텍스트 -> 임베딩 벡터
- 주가 -> 가격, 거래량, 수익률 시계열
- 고객 정보 -> 나이, 구매액, 방문횟수

이 데이터를 바탕으로 모델은 예측을 만들고, 실제 정답과 비교한 뒤, 오차를 줄이는 방향으로 계속 수정된다.

## 4. 개발자가 가장 먼저 잡아야 할 AI/ML 흐름

AI/ML을 처음 배울 때는 수식을 먼저 보기보다, 아래 흐름을 먼저 이해하는 것이 좋다.

1. 입력 데이터(feature)를 넣는다.
2. 모델이 예측값(prediction)을 만든다.
3. 실제값(label, target)과 비교한다.
4. 오차(loss)를 계산한다.
5. 미분(gradient)으로 어느 방향으로 수정할지 찾는다.
6. 가중치(weight)를 조금 바꾼다.
7. 반복한다.

즉:

예측 -> 오차 계산 -> 수정 -> 반복

이 구조가 선형회귀부터 딥러닝까지 공통 뼈대다.

---
### 📈 회귀 분석(Regression Analysis) 학습 노트

#### 1. '회귀(回歸)'라는 용어의 유래
*   **의미**: "본래의 자리(평균)로 돌아가다"
*   **유래**: 영국의 유전학자 프랜시스 골턴이 부모와 자식의 키 상관관계를 연구하며 사용. 
*   **핵심**: 아주 큰 부모의 자식은 부모보다 작아지고, 아주 작은 부모의 자식은 부모보다 커지며 인류의 평균 키로 수렴하려는 경향을 관찰하여 명명되었습니다.

---

#### 2. 선형 회귀 vs 로지스틱 회귀 비교


| 구분 | 선형 회귀 (Linear) | 로지스틱 회귀 (Logistic) |
| :--- | :--- | :--- |
| **목적** | **연속적인 숫자 예측** (Regression) | **범주 분류** (Classification) |
| **결과값** | 모든 실수 (예: 집값, 온도, 매출) | 0 ~ 1 사이의 확률 (예: 85% 확률로 합격) |
| **그래프 형태** | 직선 ($y = ax + b$) | S자 곡선 (Sigmoid Function 적용) |
| **활용 사례** | 내일의 기온 예측, 주가 예측 | 스팸 메일 분류, 질병 진단 여부 |

---

#### 3. 요점 정리
1. **선형 회귀**는 데이터 사이의 관계를 가장 잘 설명하는 **'직선'**을 찾는 과정입니다.
2. **로지스틱 회귀**는 이름은 회귀지만 실질적으로는 **'분류'**를 위해 사용됩니다.
3. 결과가 **숫자**면 선형, **OX 선택**이면 로지스틱을 선택합니다.

### 5.2 선형회귀

- 한글: 선형회귀
- 한자: 線形回歸
- 영어: Linear Regression

대표식:

y_hat = w x + b

의미:
- x: 입력값
- y_hat: 예측값
- w: 기울기(영향력)
- b: 절편(기본값)

개발자식 해석:
입력값에 가중치를 곱하고, 기본값을 더해서 결과를 예측하는 가장 기본 모델.

### 5.3 기울기

- 한글: 기울기
- 영어: Slope, Gradient

뜻:
값이 얼마나 가파르게 변하는가.

쉬운 말:
입력이 1 늘어날 때 결과가 얼마나 바뀌는가.

주의:
- 직선의 기울기 맥락에서는 slope
- 최적화 맥락에서는 gradient

### 5.4 오차

- 한글: 오차
- 한자: 誤差
- 영어: Error

뜻:
실제값과 예측값의 차이.

식:

error = y - y_hat

쉬운 말:
"얼마나 틀렸는가"

### 5.5 오차함수 / 손실함수

- 한글: 오차함수, 손실함수
- 영어: Cost Function, Loss Function

뜻:
모델이 전체적으로 얼마나 틀렸는지 계산하는 함수.

대표 예시(MSE):

MSE = (1/n) * sum((y - y_hat)^2)

왜 제곱하나:
- 음수/양수 오차가 상쇄되지 않게 하려고
- 큰 오차를 더 크게 벌주려고

### 5.6 미분

- 한글: 미분
- 한자: 微分
- 영어: Derivative, Differentiation

뜻:
입력값이 아주 조금 변할 때 결과가 얼마나 바뀌는지 보는 것.

개발자식 해석:
오차를 줄이려면 파라미터를 어느 방향으로 바꿔야 하는지 알려주는 도구.

### 5.7 경사하강법

- 한글: 경사하강법
- 한자: 傾斜下降法
- 영어: Gradient Descent

뜻:
기울기를 따라 손실이 줄어드는 방향으로 조금씩 내려가는 방법.

핵심 업데이트 식:

w = w - lr * grad

개발자식 해석:
- grad: 지금 수정 방향
- lr: 한 번에 얼마나 움직일지

즉, "오차가 줄어드는 쪽으로 weight를 조금씩 고친다"는 뜻이다.

## 18. 한 줄 요약 사전

## 19. 개발자용 공부 순서 추천

수학/통계 비전공자라면, 교과서식 순서보다 아래 흐름이 훨씬 효율적이다.

### 1단계: 용어를 코드 기준으로 익히기
- feature, target, model, loss, weight

### 2단계: 선형회귀 하나를 완전히 이해하기
- y_hat = wx + b
- 숫자 예측의 기본 구조 익히기

### 3단계: 오차와 미분의 의미 이해하기
- 왜 loss를 줄이는지
- gradient가 왜 필요한지

### 4단계: 분류로 확장하기
- logistic regression
- 확률과 decision threshold 이해하기

### 5단계: 딥러닝 용어로 확장하기
- layer, activation, backpropagation

핵심은 "수학 그 자체"보다, "이 수학이 코드에서 무슨 역할을 하는가"를 먼저 이해하는 것이다.

## 20. 마무리 정리

AI/ML에서 수학과 통계학은 개발자를 괴롭히기 위한 장벽이 아니다. 오히려 다음을 명확하게 만드는 도구다.

- 데이터를 숫자로 표현하게 해준다.
- 예측 규칙을 식으로 만들게 해준다.
- 틀린 정도를 정확히 재게 해준다.
- 덜 틀리도록 수정하게 해준다.

즉, 개발자 관점에서 보면 AI/ML의 수학은 결국 다음 한 줄로 정리된다.

> 입력을 숫자로 만들고, 오차를 계산하고, 더 잘 맞도록 값을 업데이트하는 계산 규칙

이 관점만 잡히면, 논문에 수식이 나와도 "완전히 다른 세계"가 아니라, 결국은 코드로 옮길 수 있는 로직으로 보이기 시작한다.

## 부록 A. 코드로 연결해서 보는 최소 예시

```python
from sklearn.linear_model import LinearRegression

X = [[1], [2], [3], [4]]
y = [60, 70, 80, 90]

model = LinearRegression()
model.fit(X, y)

print("weight(계수):", model.coef_)
print("bias(절편):", model.intercept_)
print("예측:", model.predict([[5]]))
```

이 코드에서:

- X: feature(입력 데이터)
- y: target(정답)
- LinearRegression: 알고리즘
- model: 학습된 모델 객체
- coef_: 가중치(weight)
- intercept_: 절편(bias)
- predict(): 추론(inference)

## 부록 B. 초보자 번역 규칙

AI/ML 용어가 어렵게 느껴질 때는 아래처럼 바꿔 읽으면 이해가 쉬워진다.

- 회귀 -> 숫자 예측
- 분류 -> 종류 예측
- 군집화 -> 자동 그룹핑
- 손실 -> 틀린 정도
- 최적화 -> 가장 잘 맞게 조정
- 파라미터 -> 모델 내부 숫자
- 하이퍼파라미터 -> 사람이 정하는 옵션
- 일반화 -> 실전 적응력
- 과적합 -> 외워버림

이 번역만 잘해도 입문 난이도가 크게 내려간다.
